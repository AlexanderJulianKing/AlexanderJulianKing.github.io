# Impeccable Design Context

## Design Context

### Users
- **Primary audience**: Hiring managers, recruiters, and technical interviewers evaluating Alexander as a candidate — typically arriving via a resume link or LinkedIn
- **Secondary audience**: Professional network contacts, collaborators, and colleagues browsing his work
- **Context**: Users are in evaluation mode — scanning quickly for credibility signals, then drilling into projects if interested. The site must establish trust and competence within seconds

### Brand Personality
**Scholarly, precise, warm** — academic rigor paired with approachability. The site should feel like reading a well-typeset research paper, not a flashy marketing page. Intelligence conveyed through restraint, not decoration.

### Aesthetic Direction
- **Visual tone**: Academic/editorial — inspired by research journals and literary publications
- **Typography-driven**: EB Garamond (serif) as primary typeface signals scholarly depth; Inter (sans) for metadata/labels provides contrast; JetBrains Mono for technical credibility
- **Color philosophy**: Warm, muted palette anchored by deep burgundy/russet accent (#7a2e2e). Off-white backgrounds (#fafaf8) with beige undertones — no cold grays or pure whites. Dark mode preserves warmth with rose-toned accent (#c46b6b)
- **Animation**: Subtle and purposeful — 0.2-0.6s durations, fade-ins on scroll, no gratuitous motion. Fully respects prefers-reduced-motion
- **Anti-references**: Avoid flashy SaaS landing pages, neon gradients, oversized hero images, emoji-heavy copy, or "tech bro" aesthetics. This is not a startup — it's a scientist's portfolio

### Design Principles
1. **Content is the hero** — Typography, whitespace, and hierarchy serve the writing. No decoration competes with the content itself.
2. **Warmth through restraint** — The burgundy accent, serif typeface, and off-white tones create warmth. Adding more color or ornamentation would dilute it.
3. **Respect the reader** — Justified text at 65ch, generous line-height (1.68), semantic HTML, and dark mode are acts of respect for the person reading.
4. **Subtle confidence** — Animations are smooth and understated. Hover states reveal, not shout. The design implies expertise without demanding attention.
5. **Accessibility is non-negotiable** — Target WCAG 2.1 AA. Semantic markup, ARIA labels, focus-visible styles, reduced motion support, and high-contrast color choices are baseline, not extras.

### Technical Constraints
- Static HTML/CSS/JS — no build tools, no frameworks, no dependencies
- All styles in a single embedded `<style>` tag
- CSS custom properties for theming (light/dark via prefers-color-scheme)
- 900px max-width container, 65ch content column
- Mobile breakpoint at 600px, sidenote collapse at 860px
- Print stylesheet included

- Background: `#fafaf8` / Alt: `#f3f2ef`
- Text: `#1a1a1a` / Muted: `#555` / Light: `#777`
- Accent: `#7a2e2e` / Hover: `#5c1e1e`
- Borders: `#d4d0c8` / Light: `#e8e5de`
- Cards: `#ffffff` / Card border: `#e8e5de`

**Dark Mode:**
- Background: `#141414` / Alt: `#1c1c1c`
- Text: `#d8d4cf` / Muted: `#a09a90` / Light: `#7a756d`
- Accent: `#c46b6b` / Hover: `#d98e8e`

- Cards: `#1a1a1a` / Card border: `#2a2a2a`

### Typography Tokens (Reference)
- Serif: `'EB Garamond', Georgia, serif` (body, headings)
- Sans: `'Inter', system-ui, sans-serif` (labels, tags, metadata)
- Mono: `'JetBrains Mono', monospace` (code)
- Base size: 18px (16px mobile)
- Line-height: 1.68 (body), 1.55 (sidenotes), 1.6 (bibliography)
